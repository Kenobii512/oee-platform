# FAZ G8 — Senaryo Kütüphanesi

Amaç: Demoda müşterinin acısına göre seçilen, her biri farklı bir kaybı baskın gösteren hazır senaryolar.
Düşük maliyet, yüksek anlatı. Simülatör parametrik olduğundan her senaryo = baseline'ın **tek bloğunun**
itilmesi. **Kalibrasyon TDD ile:** her senaryonun baskın kaybını bir test sabitler; parametreler test
yeşil olana dek ayarlanır.

**Senaryo seti (6 + 1 opsiyonel) ve hedef baskın kayıp:**

| id | Anlatı | İtilen blok | Hedef baskın kayıp (TL) |
|----|--------|-------------|-------------------------|
| `baseline` | Normal hafta | — (mevcut) | DOWNTIME (~en üst) |
| `breakdown_storm` | Arıza fırtınası | `failures` MTBF↓ MTTR↑ | DOWNTIME |
| `microstop_plague` | Mikro duruş salgını | `microstops` interval↓ | MICROSTOP |
| `speed_bottleneck` | Hız kaybı / darboğaz | `speed_loss` factor↑ | **SPEED_LOSS (gizli kanal)** |
| `fill_problem` | Doluluk problemi | `fill_loss` mean↓ | FILL_LOSS |
| `redo_crisis` | Redo/kalite krizi | `quality` sigma↑ | QUALITY_REDO |
| `world_class` *(ops.)* | İyi hafta / hedef | tüm kayıplar↓ | yok (OEE ~%85) |

```python
"""GET /scenarios -> katalog; POST /scenarios/{id}/activate -> o senaryonun verisini ingest et.

Aktivasyon, mevcut loader'ı kullanır (ground_truth firewall'u korunur). data_dir backend rootuna görece.
"""
from __future__ import annotations

from pathlib import Path

from fastapi import APIRouter, HTTPException, Request

from app.config import load_scenario_catalog
from app.ingest.loader import load_csv_dir

router = APIRouter()

_BACKEND_ROOT = Path(__file__).resolve().parents[2]


@router.get("/scenarios")
def list_scenarios(request: Request) -> dict:
    cat = load_scenario_catalog(request.app.state.config.scenario_config_path)
    return {"scenarios": [s.__dict__ for s in cat]}


@router.post("/scenarios/{scenario_id}/activate")
def activate_scenario(scenario_id: str, request: Request) -> dict:
    cat = {s.id: s for s in load_scenario_catalog(request.app.state.config.scenario_config_path)}
    info = cat.get(scenario_id)
    if info is None:
        raise HTTPException(status_code=404, detail=f"bilinmeyen senaryo: {scenario_id}")
    data_dir = (_BACKEND_ROOT / info.data_dir).resolve()
    if not data_dir.is_dir():
        raise HTTPException(status_code=404, detail=f"veri yok: {info.data_dir}")
    report = load_csv_dir(data_dir, request.app.state.repo)
    return {"activated": scenario_id, "ingest": getattr(report, "__dict__", {"ok": True})}
```
> Not: `load_csv_dir` dönüş tipini mevcut imzaya göre uyarla (G2/G4'te `LoadReport`). Dönen değer
> JSON-serileştirilebilir olmalı; gerekiyorsa sadeleştir.

# FAZ G7 — Hızlandırılmış Replay

Amaç: Seçili senaryonun bir haftalık üretimini ~10 dakikada **canlı** izlet. Backend bir **sanal saat**
ile ilerler ve her tick'te "şimdiye kadar" snapshot'ı **mevcut analytics ile** hesaplar; **SSE** ile push

**Önemli (dürüst) tasarım notu — G4.1 sınırı:** `from/to` yalnız events'e uygulanır (bilinen sınır).
Bu yüzden replay ilerledikçe **Availability, kayıp-zaman kanalları ve TL Pareto** büyür (asıl dinamik
budur); **P/Q dönem-geneli** kalır. Canlı görünüm bu yüzden **biriken duruş/TL** ve **düşen OEE/A**
üstüne kurulur — en güçlü ve dürüst anlatı. (Dönem-doğru P/Q, Dalga 3 G4.1 işidir.)

## Task G7.1: Replay motoru (sanal saat + snapshot)

**Files:** `oee-platform/backend/app/analytics/replay.py`

- [ ] **Step 1: Snapshot üreticisini yaz** — repo + config alır, olay zaman aralığını N adıma böler, her
  adımda `to=T` ile mevcut analitiği koşar. (İmzaları gerçek fonksiyonlara göre uyarla:
  `compute_oee`, `extract_loss_tree`, `to_tl`, `generate_recommendations`.)

```python
"""Hızlandırılmış replay: olayları zaman-sıralı 'şimdiye kadar' pencerelerinde özetler.

Mevcut analytics yeniden kullanılır (yeni iş kuralı YOK). FIREWALL: ground_truth ALMAZ.
G4.1 sınırı: pencere yalnız events'e (A + kayıp-zaman + TL dinamik; P/Q dönem-geneli).
"""
from __future__ import annotations

from typing import Iterator

from app.analytics.cost import to_tl
from app.analytics.loss_tree import extract_loss_tree
from app.analytics.oee import compute_oee
from app.analytics.recommend import RatioGainEstimator, generate_recommendations


def time_steps(timestamps: list[str], n_steps: int) -> list[str]:
    """Olay zaman damgalarından artan N kesim noktası (ilk→son)."""
    if not timestamps:
        return []
    ts = sorted(timestamps)
    lo, hi = ts[0], ts[-1]
    # n_steps eşit aralık; basit: indeks tabanlı bölme (determinist).
    idx = [min(len(ts) - 1, round((i + 1) * (len(ts) - 1) / n_steps)) for i in range(n_steps)]
    return [ts[j] for j in idx]


def snapshot_at(repo, line, costs, rec_cfg, to: str | None) -> dict:
    """Tek 'şimdiye kadar' snapshot'ı (to dahil)."""
    events = repo.fetch_events(None, to)
    production = repo.fetch_production()
    oee = compute_oee(events, production, line)
    tree = extract_loss_tree(events, production, line)
    cost = to_tl(tree, costs)
    recs = generate_recommendations(cost, events, rec_cfg, RatioGainEstimator(rec_cfg))
    return {
        "to": to,
        "oee": {"oee": oee.oee, "availability": oee.availability,
                "performance": oee.performance, "quality": oee.quality},
        "cost": cost,
        "total_estimated_gain_tl": sum(r["estimated_gain_tl"] for r in recs),
        "event_count": len(events),
    }


def iter_snapshots(repo, line, costs, rec_cfg, n_steps: int = 60) -> Iterator[dict]:
    """N adımlı 'şimdiye kadar' snapshot dizisi (zaman ilerledikçe büyüyen pencere)."""
    all_events = repo.fetch_events(None, None)
    stamps = [e["timestamp"] for e in all_events if e.get("timestamp")]
    for cut in time_steps(stamps, n_steps):
        yield snapshot_at(repo, line, costs, rec_cfg, cut)
```
> Not: `repo.fetch_events`/`fetch_production` ve `compute_oee`/`extract_loss_tree` imzaları G3/G4'tekiyle
> aynı olmalı; cost/recommend G11/G9'dan. Alan adları (`oee.availability` vb.) gerçek dönüş tipine göre
> uyarlanır.

## Task G7.3: SSE endpoint (GET /replay/stream)

**Files:** `oee-platform/backend/app/api/replay_routes.py`, `oee-platform/backend/app/main.py`
**Test:** `oee-platform/backend/tests/test_replay_stream.py`

- [ ] **Step 1: Endpoint yaz** — senaryoyu aktive et, snapshot'ları SSE olarak yayınla; `speed` gerçek
  zamanlı temposu ayarlar.

```python
"""GET /replay/stream?scenario=&speed=&steps= -> snapshot'ları SSE ile yayınlar.

Senaryoyu aktive eder (firewall: ground_truth ingest edilmez), iter_snapshots'u
gerçek-zaman temposuyla (tick = base/speed) push eder. media_type text/event-stream.
"""
from __future__ import annotations

import asyncio
import json

from fastapi import APIRouter, HTTPException, Query, Request
from fastapi.responses import StreamingResponse

from app.analytics.replay import iter_snapshots
from app.config import load_cost_config, load_line_definition, load_recommend_config, load_scenario_catalog
from app.ingest.loader import load_csv_dir
from app.api.scenario_routes import _BACKEND_ROOT

router = APIRouter()


@router.get("/replay/stream")
async def replay_stream(
    request: Request,
    scenario: str = Query(...),
    speed: float = Query(1.0, gt=0),
    steps: int = Query(60, gt=0, le=600),
) -> StreamingResponse:
    cfg = request.app.state.config
    cat = {s.id: s for s in load_scenario_catalog(cfg.scenario_config_path)}
    info = cat.get(scenario)
    if info is None:
        raise HTTPException(404, f"bilinmeyen senaryo: {scenario}")
    repo = request.app.state.repo
    load_csv_dir((_BACKEND_ROOT / info.data_dir).resolve(), repo)
    line = load_line_definition(cfg.line_config_path)
    costs = load_cost_config(cfg.cost_config_path)
    rc = load_recommend_config(cfg.recommend_config_path)
    base_tick = 0.2  # saniye/snapshot @ speed=1

    async def gen():
        for snap in iter_snapshots(repo, line, costs, rc, n_steps=steps):
            yield f"data: {json.dumps(snap)}\n\n"
            await asyncio.sleep(base_tick / speed)
        yield "event: done\ndata: {}\n\n"

    return StreamingResponse(gen(), media_type="text/event-stream")
```
> Eşzamanlılık: repo tek bağlantı + kilit. iter_snapshots senkron repo çağırır; uzun stream'de event
> loop'u bloklamamak için snapshot üretimini `asyncio.to_thread` ile sarmak gerekebilir (gözlemle, gerekirse uygula).
